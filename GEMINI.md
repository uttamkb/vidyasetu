# Gemini AI Context Instructions

As the Gemini AI Assistant (Antigravity), follow these instructions to establish and maintain accurate project context when starting any task in the **VidyaSetu** repository.

## 1. Primary Source of Truth
Your first step should always be to review the project's foundation.
- **Action:** Use the `view_file` tool to read `AGENT.md` in the root directory.
- **Why:** It contains the project's vision, tech stack (Next.js 16, Prisma v7, Neon DB, Auth.js v5, Inngest), core features, database schema, and page routes.

## 2. Check Knowledge Items (KIs)
Before writing any code or doing independent research:
- Review the **KI summaries** provided at the start of your conversation.
- If any KI matches your task (e.g., adding logging, background jobs, or DB schema), read its artifact path.
- KIs contain established repository patterns and past decisions. Do not skip this!

## 3. Leverage the Knowledge Graph
To understand code relationships, dependencies, and file structures efficiently:
- Look for the `graphify-out/` directory.
- This directory contains the latest code graph generated by the project's `graphify` workflow. Use it to navigate the codebase architecture without blindly searching.

## 4. Activate Relevant Skills
VidyaSetu uses specialized AI skills to enforce quality and standards. Depending on the user's request, use `view_file` to read the relevant skill's `SKILL.md` before proceeding:
- **Architecture & Planning:** `.agent/skills/architect-nextjs-ai/SKILL.md`
- **UI/UX & Frontend Design:** `.agent/skills/ux-designer/SKILL.md`
- **Coding & Bug Fixing:** `.agent/skills/claude/SKILL.md`
- **Infrastructure & Deployment:** `.agent/skills/devops/SKILL.md` or `.agent/skills/devops-engineer/SKILL.md`
- **Study Material/Content:** `.agent/skills/academic-notes-curator/SKILL.md`

## 5. Core Development Guidelines
- **Framework:** Strictly follow Next.js App Router conventions. In Next.js 16, **ALWAYS** use `src/proxy.ts` for route protection; **NEVER** create or use `middleware.ts`.
- **Styling:** Use Vanilla CSS + shadcn/ui. Ensure premium, vibrant, and accessible high-contrast designs. Avoid generic styling.
- **Database:** Use Prisma. Be mindful of race conditions, foreign key constraints, and Wasm-based Neon adapters.
- **Async Operations:** Rely on **Inngest** for background pipelines, queuing, and evaluation jobs.
- **AI Strategy:** Keep prompts highly structured, enforce schema constraints, and leverage local fallback hierarchies (School → District → State → National).

## 6. Planning Mode
- For complex, ambiguous, or architectural changes, enter **Planning Mode**.
- Create an `implementation_plan.md` artifact, outline the steps, and wait for user approval before modifying code.
- Track execution with a `task.md` checklist and summarize changes in a `walkthrough.md`.

*Always ensure your work aligns with VidyaSetu's goal of being a Cognitive Coach & Adaptive Mastery System for CBSE Students.*
